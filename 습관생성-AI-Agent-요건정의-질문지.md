# 습관 생성 AI Agent 요건 정의 질문지

## 📋 프로젝트 개요

### 1. 프로젝트 목적 및 비전
- **Q1-1**: 이 AI agent의 최종 목표는 무엇인가요?
  → **답변**: 사용자가 원하는 습관을 형성하도록 도와주고, 개인 맞춤형 습관 계획을 수립하며, 지속적인 추적 및 모니터링을 통해 습관 형성을 지원하는 것
  
- **Q1-2**: 이 프로젝트를 통해 해결하고 싶은 문제는 무엇인가요?
  → **답변**: 
  - 현재 습관 형성에서 겪는 어려움: 동기 부족, 계획 수립의 어려움, 실패 시 재시작의 어려움, 개인화된 조언 부족
  - 기존 습관 앱/서비스의 한계점: 단순 체크리스트에 그침, 실패 시 대안 제시 부족, 개인화된 피드백 부재, AI 기반 지능형 조언 부재

- **Q1-3**: 타겟 사용자는 누구인가요?
  → **답변**: 개인 사용자 (20-40대), 습관 형성에 관심이 있거나 어려움을 겪는 사람들, 자기계발을 추구하는 사람들

---

## 🎯 핵심 기능

### 2. 습관 생성 및 관리
- **Q2-1**: AI agent가 습관을 어떻게 생성하나요?
  → **답변**: 둘 다 지원. 사용자가 직접 입력할 수 있고, AI가 대화를 통해 목표를 파악하여 습관을 제안할 수도 있음

- **Q2-2**: 습관 생성 시 어떤 정보가 필요한가요?
  → **답변**: 
  - 습관 이름 (필수)
  - 목표 (예: 하루 30분 운동) (필수)
  - 빈도 (매일, 주 3회 등) (필수)
  - 기간 (30일, 90일 등) (선택)
  - 카테고리 (운동, 독서, 건강 등) (선택)
  - 알림 시간 (선택)

- **Q2-3**: 습관의 종류는 어떤 것들이 있나요?
  → **답변**: 
  - 운동, 독서, 명상, 건강, 학습, 업무 등 카테고리 분류 제공
  - 습관 간 연관성은 Phase 2에서 고려 (예: 운동 → 수면 개선)

### 3. AI 기능
- **Q3-1**: AI agent의 핵심 역할은 무엇인가요?
  → **답변**: 
  - 습관 계획 수립 조언 (핵심)
  - 진행 상황 분석 및 피드백 (핵심)
  - 동기부여 메시지 제공 (핵심)
  - 습관 실패 시 대안 제시 (핵심)
  - 일일/주간 리포트 생성 (선택)

- **Q3-2**: AI가 어떤 방식으로 상호작용하나요?
  → **답변**: 채팅 형태가 주요 인터랙션 방식. 추가로 일일 리포트와 대시보드 분석 제공

- **Q3-3**: AI의 개인화 수준은 어느 정도인가요?
  → **답변**: 사용자 데이터 기반 맞춤형 조언. 사용자의 습관 수행 기록, 성공/실패 패턴을 분석하여 개인화된 피드백 제공

### 4. 습관 추적 및 모니터링
- **Q4-1**: 사용자가 습관 수행을 어떻게 기록하나요?
  → **답변**: 
  - 체크인 (완료/미완료) (MVP 필수)
  - 수치 입력 (예: 30분, 5km) (MVP 필수)
  - 사진/일기 등 증거 제출 (Phase 2)
  - 자동 추적 (앱 연동 등) (Phase 3)

- **Q4-2**: 추적 데이터는 어떻게 활용되나요?
  → **답변**: 
  - 진행률 시각화 (필수)
  - 통계 분석 (필수)
  - AI 분석 입력 (필수)
  - 공유/소셜 기능 (Phase 3)

- **Q4-3**: 습관 실패 시 어떻게 처리하나요?
  → **답변**: 실패 기록 저장 + AI가 원인 분석 및 조언 제공. 연속 실패 시 자동으로 계획 조정 제안 (Phase 2)

## 💬 사용자 경험

### 5. 인터랙션 방식
- **Q5-1**: 사용자와 AI agent의 주요 인터랙션 방식은?
  → **답변**: 텍스트 채팅이 주요 방식. 빠른 선택을 위한 버튼/선택지도 함께 제공 (혼합형)

- **Q5-2**: 대화 톤은 어떤 스타일인가요?
  → **답변**: 친근하고 격려하는 코치 스타일. 사용자가 설정 가능하도록 옵션 제공 (Phase 2)

- **Q5-3**: 알림 및 리마인더는 어떻게 제공하나요?
  → **답변**: 브라우저 알림 (웹) + 인앱 알림. 사용자가 알림 시간 및 빈도 설정 가능

### 6. 온보딩 및 초기 설정
- **Q6-1**: 첫 사용 시 어떤 정보를 수집하나요?
  → **답변**: 
  - 목표 습관 (필수)
  - 현재 라이프스타일 (선택)
  - 과거 습관 형성 경험 (선택)
  - 선호하는 동기부여 방식 (선택)

- **Q6-2**: 온보딩 프로세스는 어떻게 진행되나요?
  → **답변**: 대화형 인터뷰 (AI와의 채팅) + 빠른 시작 옵션 제공. 사용자가 선택 가능

## 🛠 기술 스택 및 아키텍처

### 7. 기술 선택
- **Q7-1**: 어떤 플랫폼으로 개발하나요?
  → **답변**: 웹 앱 (React)으로 시작. 반응형 디자인으로 모바일도 지원. 향후 네이티브 앱 고려 (Phase 3)

- **Q7-2**: AI 모델은 어떻게 사용하나요?
  → **답변**: OpenAI GPT API 사용 (gpt-4o-mini 또는 gpt-4o). 하이브리드 방식 (규칙 기반 로직 + LLM)으로 비용 최적화

- **Q7-3**: 데이터 저장은 어떻게 하나요?
  → **답변**: 혼합형. 로컬 저장 (IndexedDB) + 클라우드 DB (Supabase). 사용자 선택에 따라 로컬만 사용 가능

### 8. 아키텍처
- **Q8-1**: 시스템 구조는 어떻게 설계하나요?
  → **답변**: 서버리스 아키텍처 (Supabase Functions + Vercel/Netlify). MVP 단계에서는 모노리식으로 시작

- **Q8-2**: 실시간 기능이 필요한가요?
  → **답변**: 실시간 채팅 (WebSocket 또는 Server-Sent Events). 실시간 알림은 브라우저 알림 API 사용

## 📊 데이터 및 학습

### 9. 데이터 수집 및 활용
- **Q9-1**: 어떤 데이터를 수집하나요?
  → **답변**: 
  - 습관 수행 기록 (필수)
  - 사용자 피드백 (선택)
  - 성공/실패 패턴 (필수)
  - 개인 정보는 최소한만 수집 (이메일, 닉네임)

- **Q9-2**: 데이터 프라이버시는 어떻게 보장하나요?
  → **답변**: 
  - 사용자 동의 하에만 수집
  - 로컬 저장 우선 (선택 가능)
  - 데이터 암호화 저장
  - GDPR 등 규정 준수

- **Q9-3**: AI 모델 학습은 어떻게 하나요?
  → **답변**: 사용자별 개인화는 프롬프트 엔지니어링과 RAG(Retrieval-Augmented Generation)로 구현. 전역 모델 업데이트는 Phase 3에서 고려

### 10. 분석 및 인사이트
- **Q10-1**: 어떤 인사이트를 제공하나요?
  → **답변**: 
  - 습관 성공률 통계 (필수)
  - 패턴 분석 (요일별, 시간대별) (필수)
  - 습관 간 상관관계 (Phase 2)
  - 목표 달성 예측 (Phase 2)

- **Q10-2**: 시각화는 어떤 형태인가요?
  → **답변**: 
  - 차트/그래프 (성공률, 트렌드)
  - 캘린더 뷰 (히트맵 형태)
  - 진행률 바 (각 습관별)
  - 히트맵 (요일별/시간대별 패턴)

## 🎨 UI/UX 디자인

### 11. 디자인 방향
- **Q11-1**: 디자인 스타일은 어떤가요?
  → **답변**: 미니멀리스트 스타일. 다크모드 지원. 색상은 부드럽고 편안한 톤 사용

- **Q11-2**: 주요 화면 구성은?
  → **답변**: 
  - 홈/대시보드 (오늘의 습관, 진행률 요약)
  - 습관 목록 (전체 습관 관리)
  - AI 채팅 화면 (AI와의 대화)
  - 통계/분석 화면 (인사이트 및 시각화)
  - 설정 화면 (알림, 프로필 등)

- **Q11-3**: 접근성 고려사항은?
  → **답변**: 
  - 시각 장애인 지원 (스크린 리더, ARIA 라벨)
  - 다국어 지원 (한국어, 영어) - Phase 2
  - 다양한 디바이스 대응 (반응형 디자인)

## 🚀 배포 및 운영

### 12. 배포 전략
- **Q12-1**: 배포 환경은?
  → **답변**: 베타 테스트로 시작. 이후 프로덕션 배포. 오픈소스 공개 고려 (Phase 2)

- **Q12-2**: 사용자 규모 예상은?
  → **답변**: 소규모 (수십 명)로 시작. 점진적으로 확장

- **Q12-3**: 비용 및 리소스 제약은?
  → **답변**: 무료 티어 우선 활용 (Supabase, Vercel). OpenAI API는 사용량 기반 과금. 비용 모니터링 필수

### 13. 유지보수 및 개선
- **Q13-1**: 지속적인 개선 계획은?
  → **답변**: 
  - 사용자 피드백 수집 (인앱 피드백 폼)
  - 정기 업데이트 (월 1회)
  - 신기능 추가 로드맵 (Phase별 계획)
  - A/B 테스트는 Phase 2 이후 고려

- **Q13-2**: 모니터링 및 로깅은?
  → **답변**: 
  - 에러 추적 (Sentry 또는 유사 서비스)
  - 사용량 분석 (Supabase Analytics)
  - 성능 모니터링 (Vercel Analytics)
  - 사용자 행동 분석 (Google Analytics 또는 Mixpanel)

## 🎯 우선순위 및 범위

### 14. MVP (Minimum Viable Product)
- **Q14-1**: MVP에 포함할 최소 기능은?
  → **답변**: 
  - 필수: 
    - 습관 생성/수정/삭제
    - 습관 체크인 (완료/미완료, 수치 입력)
    - AI 채팅 (습관 조언, 피드백)
    - 기본 통계 (성공률, 진행률)
    - 캘린더 뷰 (히트맵)
    - 알림 설정
  - 선택: 
    - 상세 통계 (요일별, 시간대별)
    - 습관 카테고리

- **Q14-2**: MVP 이후 추가할 기능은?
  → **답변**: 
  - Phase 2: 
    - 습관 간 상관관계 분석
    - 목표 달성 예측
    - 사진/일기 첨부
    - 다국어 지원
    - 대화 톤 커스터마이징
  - Phase 3: 
    - 자동 추적 (앱 연동)
    - 소셜 기능 (공유, 챌린지)
    - 네이티브 모바일 앱
    - 고급 AI 분석

- **Q14-3**: 개발 일정은?
  → **답변**: 
  - 시작일: 2026-01-XX
  - MVP 목표일: 2026-03-XX (약 8-10주)
  - 최종 목표일: 2026-06-XX (Phase 2 완료)

## 💡 추가 고려사항

### 15. 기타
- **Q15-1**: 참고하고 싶은 기존 서비스나 앱이 있나요?
  → **답변**: 
  - Habitica (게이미피케이션 요소)
  - Streaks (심플한 UI/UX)
  - Notion (유연한 데이터 구조)
  - ChatGPT (대화형 인터페이스)

- **Q15-2**: 특별히 고려해야 할 제약사항이나 요구사항이 있나요?
  → **답변**: 
  - 기술적 제약: OpenAI API 비용 관리, 무료 티어 한도
  - 법적/규제 요구사항: GDPR 준수, 개인정보 보호
  - 비즈니스 요구사항: 오픈소스 라이선스 고려

- **Q15-3**: 성공 지표는 무엇인가요?
  → **답변**: 
  - 사용자 수 (DAU/MAU)
  - 습관 성공률 (사용자 평균 습관 달성률)
  - 사용자 만족도 (NPS, 앱 리뷰)
  - AI 채팅 사용 빈도
  - 습관 지속 기간 (평균 습관 유지 일수)
