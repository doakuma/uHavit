- **프로젝트명**: 습관 생성 AI Agent
- **버전**: 1.0
- **작성일**: 2026-01-XX
- **최종 수정일**: 2026-01-XX

---

## 1. 프로젝트 개요

### 1.1 프로젝트 목적
사용자가 원하는 습관을 형성하도록 도와주고, 개인 맞춤형 습관 계획을 수립하며, 지속적인 추적 및 모니터링을 통해 습관 형성을 지원하는 AI 기반 웹 애플리케이션입니다.

### 1.2 해결하고자 하는 문제
- **현재 습관 형성에서 겪는 어려움**
  - 동기 부족으로 인한 지속성 문제
  - 효과적인 계획 수립의 어려움
  - 실패 시 재시작의 어려움
  - 개인화된 조언 및 피드백 부족

- **기존 습관 앱/서비스의 한계점**
  - 단순 체크리스트에 그치는 기능
  - 실패 시 대안 제시 부족
  - 개인화된 피드백 부재
  - AI 기반 지능형 조언 부재

### 1.3 타겟 사용자
- **주요 타겟**: 20-40대 개인 사용자
- **특성**: 습관 형성에 관심이 있거나 어려움을 겪는 사람들, 자기계발을 추구하는 사람들

## 2. 핵심 기능

### 2.1 습관 생성 및 관리
- **습관 생성 방식**
  - 사용자 직접 입력
  - AI 대화를 통한 습관 제안 (대화형 인터뷰)
  - 둘 다 지원

- **습관 생성 시 수집 정보**
  - 습관 이름 (필수)
  - 목표 (예: 하루 30분 운동) (필수)
  - 빈도 (매일, 주 3회 등) (필수)
  - 기간 (30일, 90일 등) (선택)
  - 카테고리 (운동, 독서, 건강 등) (선택)
  - 알림 시간 (선택)

- **습관 종류 및 분류**
  - 카테고리: 운동, 독서, 명상, 건강, 학습, 업무 등
  - 습관 간 연관성 분석은 Phase 2에서 구현

### 2.2 AI Agent 기능
- **핵심 역할**
  - 습관 계획 수립 조언
  - 진행 상황 분석 및 피드백
  - 동기부여 메시지 제공
  - 습관 실패 시 대안 제시
  - 일일/주간 리포트 생성 (선택)

- **상호작용 방식**
  - 채팅 형태가 주요 인터랙션 방식
  - 일일 리포트 및 대시보드 분석 제공

- **개인화 수준**
  - 사용자 데이터 기반 맞춤형 조언
  - 사용자의 습관 수행 기록, 성공/실패 패턴 분석
  - 개인화된 피드백 제공

### 2.3 습관 추적 및 모니터링
- **기록 방식**
  - 체크인 (완료/미완료) - MVP 필수
  - 수치 입력 (예: 30분, 5km) - MVP 필수
  - 사진/일기 등 증거 제출 - Phase 2
  - 자동 추적 (앱 연동 등) - Phase 3

- **데이터 활용**
  - 진행률 시각화
  - 통계 분석
  - AI 분석 입력
  - 공유/소셜 기능 - Phase 3

- **실패 처리**
  - 실패 기록 저장
  - AI 원인 분석 및 조언 제공
  - 연속 실패 시 자동 계획 조정 제안 - Phase 2

## 3. 사용자 경험

### 3.1 인터랙션 방식
- **주요 방식**: 텍스트 채팅
- **보조 방식**: 빠른 선택을 위한 버튼/선택지 제공 (혼합형)

### 3.2 대화 톤
- **기본 스타일**: 친근하고 격려하는 코치 스타일
- **커스터마이징**: 사용자가 설정 가능하도록 옵션 제공 - Phase 2

### 3.3 알림 및 리마인더
- **제공 방식**: 브라우저 알림 (웹) + 인앱 알림
- **설정**: 사용자가 알림 시간 및 빈도 설정 가능

### 3.4 온보딩
- **수집 정보**
  - 목표 습관 (필수)
  - 현재 라이프스타일 (선택)
  - 과거 습관 형성 경험 (선택)
  - 선호하는 동기부여 방식 (선택)

- **프로세스**: 대화형 인터뷰 (AI와의 채팅) + 빠른 시작 옵션 제공

## 4. 기술 스택

### 4.1 플랫폼
- **MVP**: 웹 앱 (React)
- **디자인**: 반응형 디자인으로 모바일 지원
- **향후**: 네이티브 모바일 앱 - Phase 3

### 4.2 AI 모델
- **사용 모델**: OpenAI GPT API (gpt-4o-mini 또는 gpt-4o)
- **구현 방식**: 하이브리드 (규칙 기반 로직 + LLM)로 비용 최적화

### 4.3 데이터 저장
- **방식**: 혼합형
  - 로컬 저장 (IndexedDB)
  - 클라우드 DB (Supabase)
- **선택권**: 사용자가 로컬만 사용 가능하도록 선택 제공

### 4.4 아키텍처
- **구조**: 서버리스 아키텍처 (Supabase Functions + Vercel/Netlify)
- **MVP**: 모노리식으로 시작

### 4.5 실시간 기능
- **채팅**: WebSocket 또는 Server-Sent Events
- **알림**: 브라우저 알림 API

## 5. 데이터 및 프라이버시

### 5.1 데이터 수집
- **필수 데이터**
  - 습관 수행 기록
  - 성공/실패 패턴
- **선택 데이터**
  - 사용자 피드백
  - 최소한의 개인 정보 (이메일, 닉네임)

### 5.2 프라이버시 보장
- 사용자 동의 하에만 수집
- 로컬 저장 우선 (선택 가능)
- 데이터 암호화 저장
- GDPR 등 규정 준수

### 5.3 AI 모델 학습
- **MVP**: 사용자별 개인화는 프롬프트 엔지니어링과 RAG로 구현
- **향후**: 전역 모델 업데이트는 Phase 3에서 고려
